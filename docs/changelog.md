## 2026-04-01

### 提供時間の記録機能を追加（日々の記録）
- 日々の記録の送迎時間カードに「提供時間」セクション（開始・終了）を追加
- お迎え到着時間を入力すると、提供開始時間に同じ時間を自動入力
- 提供終了時間は通常日=16:30、学校休日=16:00 をデフォルトで自動入力
- 学校休日かどうかは子どもの学校休日設定（child_school_holidays）を参照
- 通知設定ページ（/settings/notifications）でデフォルト終了時間を変更可能
  - 「通常日の終了時間」と「学校休日の終了時間」をそれぞれ設定できる
- DB変更：`daily_attendance` に `service_start_time`, `service_end_time` を追加
- DB変更：`notification_settings` に `default_service_end_time`（デフォルト16:30）、`holiday_service_end_time`（デフォルト16:00）を追加
- マイグレーション：`supabase/migrations/053_service_times.sql`

### 無効化した利用計画が送迎スケジュールに反映される問題を修正
- 利用計画を無効化（is_active=false）しても、事前生成した usage_reservations が残り、送迎スケジュールに含まれてしまう不具合を修正
- 利用計画を無効化する際、今日以降の確定・仮予約をキャンセル状態に更新するよう変更
- 送迎スケジュール自動生成の時間補完ロジック（Step1〜3）に `is_active=true` フィルタを追加

### 出席履歴一覧から出席・欠席の切り替えを可能に
- 子どもの出席履歴ページ（/attendance/child/[childId]）の各行のバッジをクリックで出席⇔欠席を切り替えられるようになった
- クライアントコンポーネント `AttendanceStatusToggle` を新規作成
- ホバー時にツールチップで操作説明を表示

## 2026-03-XX（直近の主要実装まとめ）

### 日々の記録に送迎時間の記録を追加
- 日々の記録フォームの送迎時間カードを一番上に移動
- お迎えに行った時間・事務所に到着した時間・事務所を出た時間・自宅に到着した時間 の4項目を追加
- お迎え時間入力 → 10分後の到着時間を自動入力
- 自宅到着時間入力 → 10分前の出発時間を自動入力
- DB変更：`daily_attendance` に `pickup_departure_time`, `pickup_arrival_time`, `dropoff_departure_time`, `dropoff_arrival_time` を追加
- マイグレーション：`supabase/migrations/052_daily_attendance_transport_times.sql`

### 個別支援計画の長期目標・短期目標に5段階評価を追加
- 新規作成フォームと既存計画の編集カードに5段階の星評価（StarRating）を追加
- スタッフは閲覧時に評価を表示、管理者のみ編集可能
- DB変更：`support_plans` に `long_term_goal_rating`, `short_term_goal_rating` を追加（CHECK 1〜5）
- マイグレーション：`supabase/migrations/051_support_plan_goal_ratings.sql`

### スタッフのアクセス権限を整理
- スタッフはシフト管理・設定・国保連請求にアクセス不可
- スタッフはメイン機能（出席・記録・個別支援計画・送迎・連絡帳・お知らせ・利用状況・児童管理・書類）すべて閲覧可能
- 個別支援計画はスタッフ閲覧のみ・管理者のみ編集可能
- Middleware（`src/middleware.ts`）と各ページの `requireAdmin()` の両方で制限を実施

### LINEの送迎通知の重複送信を修正
- Vercelプロジェクトが2つ存在し、2通送信されていた問題を解消
- `vercel.json` のcronを `0 5 * * *`（JST午後2時）に1本のみ設定
- 不要なVercelプロジェクト（houkago-day-service-app）を削除
